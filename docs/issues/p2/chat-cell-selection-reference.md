# 将单元格选区添加到 AI 对话

- **优先级**：P2
- **状态**：待处理
- **范围**：前端选区交互、聊天引用协议、服务端引用校验和 Agent 上下文

当前聊天支持引用工作簿和 Sheet，但不支持将具体单元格选区作为对话引用。用户选中一段区域后，无法明确告诉模型后续请求针对哪个 Sheet 的哪些单元格，模型只能依赖自然语言描述或再次调用工具猜测范围。

应提供显式的“添加到对话”操作。用户选中区域后，将工作簿、Sheet 和 A1 范围加入聊天输入区的引用标签，例如 `销售数据 / Sheet3 / B2:E10`。发送时只提交结构化引用，不直接提交完整单元格数据；服务端负责校验工作区权限和范围有效性，Agent 再通过表格读取工具获取数据。

## 验收条件

- 选中单元格区域后可以显式添加到当前对话，并支持移除或重新选择引用；
- 引用至少包含 `workbookId`、`sheetId` 和 A1 范围，前端不提交完整工作簿或完整 Sheet 数据；
- 服务端校验引用所属工作区、Sheet 存在性和范围格式，失效引用不得静默指向其他 Sheet；
- Agent 能在上下文中明确识别工作簿、Sheet 和单元格范围，并可据此完成读取、分析和图表等操作；
- 重新加载会话后，选区引用仍能正确展示或明确标记为不可用；
- 覆盖选区提取、请求协议、服务端解析和 Agent 上下文转换测试。
