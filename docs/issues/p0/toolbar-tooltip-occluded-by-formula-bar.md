# 功能区按钮提示被公式栏遮挡

- **优先级**：P0
- **状态**：待修复
- **范围**：Web 工作簿编辑器、FortuneSheet 功能区与公式栏的层级布局
- **记录时间**：2026-07-24

## 问题

鼠标悬浮在工作簿顶部功能区按钮上时，功能提示没有显示在最上层，而是被下方的 Sheet 名称/公式栏区域遮挡。用户无法看到按钮的完整提示，功能区中的部分能力因此难以识别和使用。

## 复现

1. 打开任意工作簿编辑页面；
2. 将鼠标悬浮在功能区任意带提示的按钮上；
3. 观察提示层与公式栏、Sheet 内容区域的重叠关系。

## 影响

- 功能区按钮的 tooltip 被其他编辑区域覆盖，核心操作的可发现性和可用性下降；
- 不同窗口尺寸、滚动状态或按钮位置下可能出现不一致的遮挡；
- 继续增加功能区按钮或浮层时，层级冲突会扩大，导致更多交互反馈不可见。

## 修复方向

- 明确工作簿编辑器内功能区 tooltip、公式栏、表格画布、Sheet 标签和图表浮层的层级契约；
- 确保 tooltip 的定位上下文和 `z-index` 高于公式栏及其他非弹窗编辑区域，并在编辑器边界内正确处理溢出；
- 优先修正现有 FortuneSheet tooltip 的容器层级和父级 `overflow`，避免通过扩大单个按钮的层级进行局部打补丁；
- 在桌面和窄视口下分别验证顶部按钮、靠近左右边界按钮以及公式栏相邻按钮的提示位置。

## 验收条件

- 功能区所有带提示的按钮悬浮时，提示内容完整可见，不被公式栏、Sheet 标签、表格画布或图表覆盖；
- tooltip 不会被编辑器容器裁切，也不会遮挡触发按钮的主要操作区域；
- 在常用桌面尺寸、窄窗口和页面滚动状态下，提示层级和定位保持一致；
- 其他 FortuneSheet 菜单、确认框、图表浮层和全局弹窗的层级不回归；
- 增加针对 tooltip 层级或关键 DOM 结构的回归验证，避免后续样式调整重新引入遮挡。
